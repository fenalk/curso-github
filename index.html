Os passos para criar um repositório no github

1. "git init" comando para iniciar um repositorio/faça dentro da pasta no git bash ou cmd
2. "git add ." "git add nome-do-arquivo.formato"adicionar os arquivos
3. "git commit -m 'comentário para organização'" comitamos um arquivo
4. "git branch -M main" criar branch
5. "git remote add origin https:// ..." passa qual o repositorio que criamos no github
6. "git push -u origin main" para enviar o arquivo do nosso local para repositorio
7. "git pull" para adicionar arquivo do repositorio para o local/sincronização

Clonar repositórios do github

1. "git clone hppts://...." url do reposotótio
2. "git clone hppts://...." com ponto no final baixa apenas o arquivo para pasta


Acessando log

1. "git log" irá aparecer todos as modificações/adições etc nos arquivos.
2. pressione 'q' para sair do git log


Revertendo arquivo (Checkout)

- trazer original lá do github antes de alterar
1. "git checkout nome-do-arquivo.formato"


Ignorar arquivos (não enviar ao repositorio)

1. criar o arquivo .gitignore na raiz do projeto
2. escreva o nome dos arquivos no .gitignore os arquivos que deseja não enviar ao github


Resetando arquivos da branch

1. git reset --hard origin/main (resetar para a branch anterior, ou seja, ao último commit/push que tinha enviado ao github)
2. tem que tomar cuidado com este comando. talvez nem tudo queira que volte ao último commit/push


O que é branches?

Ideal para trabalhar em empresas. É uma maneira que o git disponibiliza para separar versões do projeto, gerenciar melhor o projeto.

Criamos um novo projeto ele inicia sempre na Branch main


Main > Pagina Home > Main (intacta) > Pagina Home Pronta > (Merge) > Main


Criando branchs

1. "git branch" para verificar quantas branches existem no projeto. * main ( * quer dizer que estou na branch main)
2. "nome_da_branch" geralmente tem um padrão de nome fornecido da branches


Mudando de branch

1. "git checkout nome_da_branch muda de branch, fica um * no nome da branch
2. "git checkout -b nome_da_branch" cria uma nova branch e automaticamente entra nela


Excluindo uma branch

1. "git branch -d nome_da_branch" irá deletar a branch. cuidado!


Unir as branches (merge)

1. "git merge nome_da_branch_que_quer_unir" irá unir branch que deseja unir com a main



Git Stash

1. "git stash" vai buscar a última versão da main
2. "git stash list" verificar as stash salvas
3. "git stash apply nome_do_id_que_quer_voltar" o nome verifica a partir de git stash list
4. "git stash show -p" verifica o que tem em cada stash (clica q para sair)

Conhecendo Tags

1. "git tag -a v1.0(versão) -m 'Primeira versão do sistema'"
2. "git tag" para verifica a versão no sistema
